# CLI Commands

## Process Management

### `nova-pm start`

Start a process or all processes from a config file.

```bash
nova-pm start <script|config> [options]
```

**Options:**
| Flag | Description | Default |
|------|-------------|---------|
| `-n, --name <name>` | Process name | Script filename |
| `-i, --instances <n>` | Number of instances | `1` |
| `--exec-mode <mode>` | `fork` or `cluster` | `fork` |
| `-w, --watch` | Watch for file changes | `false` |
| `--max-memory <size>` | Max memory (e.g., `512M`) | — |
| `--env <env>` | Environment name | — |
| `-p, --port <port>` | Application port | — |
| `--cron <pattern>` | Cron restart pattern | — |
| `--no-autorestart` | Disable auto-restart | — |
| `--interpreter <path>` | Custom interpreter | `node` |
| `--node-args <args>` | Node.js arguments | — |
| `--cwd <dir>` | Working directory | Current dir |

### `nova-pm stop`

```bash
nova-pm stop <name|id|all> [--force]
```

### `nova-pm restart`

```bash
nova-pm restart <name|id|all>
```

### `nova-pm delete`

```bash
nova-pm delete <name|id|all>
```

### `nova-pm scale`

```bash
nova-pm scale <name|id> <instances>
# Examples:
nova-pm scale api 8      # Set to 8 instances
nova-pm scale api +2     # Add 2 instances
nova-pm scale api -1     # Remove 1 instance
```

## Information

### `nova-pm list`

```bash
nova-pm list [--json]
```

### `nova-pm info`

```bash
nova-pm info <name|id> [--json]
```

### `nova-pm logs`

```bash
nova-pm logs [name|id] [options]
```

| Flag | Description | Default |
|------|-------------|---------|
| `-l, --lines <n>` | Lines to show | `50` |
| `-f, --follow` | Stream new logs | `false` |
| `--json` | Output as JSON | `false` |

### `nova-pm monit`

Real-time terminal monitoring dashboard.

```bash
nova-pm monit
```

## System

### `nova-pm ping`
Check if the daemon is running.

### `nova-pm save`
Save current process list for restoration.

### `nova-pm resurrect`
Restore previously saved process list.

### `nova-pm startup`
Generate OS startup script (systemd, launchd).

### `nova-pm dashboard`
Open the web dashboard.

### `nova-pm init`
Generate a `nova-pm.config.js` file interactively.

### `nova-pm doctor`
Diagnose NovaPM installation.
